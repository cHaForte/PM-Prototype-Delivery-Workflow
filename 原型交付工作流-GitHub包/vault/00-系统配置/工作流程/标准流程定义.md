# 标准原型交付流程（v3.0）

## 流程概要

```
用户输入需求 → 需求整理 → 确认 → 原型生成 → 迭代调整（←→用户反馈）
                                              ↓ 满意后
                                         文档撰写（三份）
                                          ↓ 确认
                                       项目完成
```

## 阶段详情

### 阶段1：需求整理
- 角色：agent-requirement（模式A）
- 输入：用户原始需求
- 输出：需求整理摘要（结构化需求描述）
- 产出路径：vault/01-需求管理/需求整理摘要/
- 特点：只整理，不生成变更单
- 确认：需要

### 阶段2：原型生成（核心阶段）

#### 子阶段A：生成初始原型
- 角色：agent-prototype（模式A）
- 输入：需求整理摘要
- 输出：HTML/CSS/JS 初始原型
- 产出路径：prototypes/[项目名]/

#### 子阶段B：迭代调整循环
- 状态：[原型调整中]
- 用户随时提出修改意见 → Agent 局部修改
- 循环直到用户说"满意"

#### 子阶段C：文档撰写（原型定稿后）
顺序产出三份文档：

| 文档 | 角色 | 输出路径 |
|------|------|---------|
| 需求变更单 | agent-requirement（模式C） | vault/01-需求管理/需求变更单/ |
| 测试大纲 | agent-qa（模式B） | vault/03-质量管理/测试大纲/ |
| 用户帮助文档 | agent-doc（模式B） | vault/04-文档管理/用户帮助文档/ |

## 迭代交互示例

```
[Orch] [当前状态: 原型调整中]
       原型路径：prototypes/xxx/index.html
       提出修改意见，或说"满意"

你：把按钮改成蓝色
[Agent] 已修改

你：加一个表单验证
[Agent] 已追加验证逻辑

你：满意
[Orch] 原型定稿，进入文档撰写...
       → 需求变更单（基于最终实现）
       → 测试大纲（基于最终原型）
       → 用户帮助文档（基于最终原型）
```

## 执行模式
默认：半自动（每阶段需用户确认，原型阶段有迭代循环）
