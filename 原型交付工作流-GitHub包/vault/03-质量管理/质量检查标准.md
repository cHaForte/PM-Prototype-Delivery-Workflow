# 质量检查标准

## 检查维度

### 1. 功能完整性（权重：高）
- 所有需求功能已实现
- 交互流程完整
- 状态覆盖（成功、失败、空、加载）
- 边界条件处理

### 2. 代码质量（权重：中）
- HTML 语义化
- CSS 使用 Flexbox/Grid
- JS 无语法错误
- 零框架依赖

### 3. 用户体验（权重：高）
- 布局合理
- 交互反馈及时
- 导航流畅
- 错误提示清晰

### 4. 响应式（权重：中）
- 移动端适配
- 桌面端适配
- 交互在多尺寸可用

## 问题分级

| 级别 | 定义 | 处理 |
|------|------|------|
| P0 | 功能缺失/严重错误 | 阻塞流程 |
| P1 | 显著影响体验 | 强烈建议修复 |
| P2 | 轻微问题 | 建议修复 |
| P3 | 优化建议 | 可选 |
